# Imagebin API Documentation

Even though this API was originally designed just for photo hosting, it allows you to host any file format on the server.

---

## 1.. General Rules & Authentication

It is mandatory to include authentication information in all requests sent to the API. Authentication is performed via the HTTP Header using the `x-api-key` key.

* **Auth Method:** API Key
* **Header Key:** `x-api-key`
* **Header Value:** `<config.yaml dosyasında belirlenen API Key>`

---

## 2. Configuration (config.yaml)

The entire operational workflow of the API is managed via the `config.yaml` file. The majority of the settings can be updated dynamically. Below are the parameters and their descriptions located in the `config.yaml` file:

| Parameter            | Data Type    | Description                                                                                                                                                                       |
| :------------------- | :----------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `api_key`            | String       | The secret key required to access the API.                                                                                                                                        |
| `domain`             | String       | The domain name or IP address where the server is hosted.                                                                                                                         |
| `url_prefix`         | String       | The URL prefix for hosted files. (e.g., `images`).<br>**Note:** This parameter cannot be changed dynamically via config reload (`/reload-config`); the service must be restarted. |
| `upload_dir`         | String       | The local folder name where files will be stored unencrypted and physically on the server. File names are kept exactly identical to the `image_id` in the system.                 |
| `allowed_extensions` | List (Array) | The list of permitted file extensions (e.g., `[.png, .jpg, .pdf]`). Any file with an extension outside of this list will be rejected and not uploaded.                            |

---

## 3. API Endpoints

### 3.1. Upload Files
Used to upload a new file to the system.

* **HTTP Method:** `POST`
* **Endpoint:** `/v1/upload`
* **Headers:**
    * `x-api-key`: `<api_key>`
* **Request Body (multipart/form-data):**
    * `image` (File): The local file to be uploaded.

#### Successful response example(`200 OK`)
```json
{
    "status": "200",
    "image_id": "f4300195-e30f-436c-aa1b-998992f8f3e3.png",
    "url": "http://api.domain.tld/images/f4300195-e30f-436c-aa1b-998992f8f3e3.png"
}

```

### 2. Delete File

Permanently deletes a file hosted on the server from both the system and the disk.

- **HTTP Method:** `POST`
    
- **Endpoint:** `/v1/delete`
    
- **Headers:**
    
    - `x-api-key`: `<api_key>`
        
- **URL Query Parameters (Params):**
    
    - `image_id` (String / Required): The unique ID given to the file during upload that you wish to delete (including its extension, e.g., `f4300195...png`).

## Succesful response example (`200 OK`)
```json
{
    "status": "200",
    "message": "Image deleted successfully"
}
```

### 3. Reload Config

Reloads the changes made to `config.yaml` into the system without shutting down and restarting the API or the server (seamlessly/without downtime).

- **HTTP Method:** `POST`
    
- **Endpoint:** `/v1/reload-config`
    
- **Headers:**
    
    - `x-api-key`: `<api_key>`
        
- **Request Body:** None.
    

#### Successful response example(`200 OK`)
```json
{
    "status": "200",
    "message": "Configuration reloaded successfully"
}
```
## 4. Storage Details

- All uploaded files are stored unencrypted (**raw**) under the directory specified in `config.yaml`.
    
- To prevent conflicts, file names on the disk are saved to be exactly identical to the unique `image_id` value generated by the API.
###  5. How to self-host?
The project is available on Docker Hub. You can pull the image by typing:

```bash
docker pull cakiejoy/imagebin-api:0306.2026.2
```

and run the Docker Compose file located in the [github.com/CakieJoy/compose-stack](https://github.com/CakieJoy/compose-stack) repository.
